# Улучшения админки Keystatic (Главная)

## Что уже сделано

### 1. Подписи элементов массивов (вместо «Item 1», «Item 2»)
Во всех массивах в `keystatic.config.ts` добавлен **itemLabel** — функция, которая задаёт текст для каждого элемента в списке:

- **Объекты с заголовком** (карточки, шаги, колонки): показывается `title` или `name` (например: «Проработка индивидуального запроса», «IT-студия»).
- **Объекты со значением и подписью** (статы Hero): показывается «Значение — Подпись».
- **Объекты с текстом** (карточки «Особенно актуально», FAQ): показывается начало текста (до 40–50 символов).
- **Массивы строк** (теги, пункты списков): показывается сам текст пункта или «Тег» / «Пункт» по умолчанию.

Так проще находить нужный элемент в списке при редактировании.

### 2. Подписи и описания секций на «Главной»
У каждой секции (Hero, Почему доверяют, CTA, Диагностика и т.д.):

- **label** — короткое название секции (как было, местами уточнено).
- **description** — подсказка, что именно редактируется в этом блоке (например: «Блок «Обсудим задачи вашей команды» с кнопками связи»).

В интерфейсе Keystatic под заголовком секции отображается описание, так проще понять, какой блок страницы ты редактируешь.

## Идеи для дальнейших улучшений

### Навигация по разделам
- Сейчас «Главная» — одна длинная форма; в Keystatic нет встроенной «оглавления» для такой формы.
- **Вариант A:** Использовать поиск по странице (Cmd+F / Ctrl+F) по названию секции (например, «Hero», «Обсудим»).
- **Вариант B:** Если в будущем в Keystatic появится навигация/якоря для полей — включить её в конфиг.
- **Вариант C:** Разнести контент по нескольким singleton’ам (например, «Главная — Hero», «Главная — Блоки»), если готовы менять структуру контента и код страницы.

### Порядок секций
Порядок блоков в схеме совпадает с порядком на странице. Если нужно, чтобы в админке чаще редактируемые блоки были выше, их можно переставить в `schema` (и при необходимости в `home.yaml`), сохраняя соответствие с вёрсткой на сайте.

### Локализация интерфейса Keystatic
Часть UI (кнопки «Add», «Delete» и т.п.) задаётся самим Keystatic. Если понадобится полная локализация, стоит посмотреть актуальную документацию по кастомизации/локализации Keystatic.

## Где что менять

- **Схема и подписи:** `keystatic.config.ts` — секция `home: singleton({ schema: { ... } })`.
- **Контент главной:** `src/content/home.yaml`.
- **Настройки сайта (меню, мессенджер и т.д.):** `src/content/siteSettings.yaml`.
